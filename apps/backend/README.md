# Airix Backend

Go REST API for Airix. Stack: Chi router, pgx, sqlc, golang-migrate, Redis, AWS S3, gRPC (for AI backend communication).

---

## Global Tools (install once, on your machine)

These are not Go packages — they are system-level binaries. Install them globally.

```bash
# 1. Go (1.22+)
# Download from https://go.dev/dl and install

# 2. Database migration CLI
go install -tags 'postgres' github.com/golang-migrate/migrate/v4/cmd/migrate@latest

# 3. Hot reload for development
go install github.com/air-verse/air@latest

# 4. sqlc — generates Go DB code from SQL queries
go install github.com/sqlc-dev/sqlc/cmd/sqlc@latest

# 5. protoc — the Protobuf compiler (BINARY, not a Go package)
# Windows: download protoc-XX.X-win64.zip from https://github.com/protocolbuffers/protobuf/releases
# Extract and add the bin/ folder to your system PATH
# Then verify: protoc --version

# 6. Go plugins for protoc (needed after installing protoc)
go install google.golang.org/protobuf/cmd/protoc-gen-go@latest
go install google.golang.org/grpc/cmd/protoc-gen-go-grpc@latest
```

Make sure `$GOPATH/bin` is in your PATH so the installed Go binaries are available as commands.

## Local Setup

```bash
# 1. Copy environment file
cp .env.example .env
# Fill in your actual values

# 2. Create the Postgres database
psql -U postgres -c "CREATE DATABASE airix;"

# 3. Run migrations
migrate -path db/migrations -database "postgres://user:password@localhost:5432/airix?sslmode=disable" up

# 4. Install Go dependencies
go mod download

# 5. Start with hot reload
air
```

Server: `http://localhost:8080`  
Health check: `GET /api/v1/health`

## Database

```bash
# Run all pending migrations
migrate -path db/migrations -database "$DATABASE_URL" up

# Rollback last migration
migrate -path db/migrations -database "$DATABASE_URL" down 1

# After editing any file in db/query/*.sql — regenerate Go DB code
sqlc generate
```

> Never manually edit files inside `internal/db/` — they are auto-generated by sqlc.

## gRPC — AI Backend Contract

The Go backend calls the Python AI backend via gRPC. The `.proto` file is the shared contract between both sides.

```bash
# After editing proto/ai/ai.proto — regenerate Go client code
protoc \
  --go_out=internal/pb/ai \
  --go_opt=paths=source_relative \
  --go-grpc_out=internal/pb/ai \
  --go-grpc_opt=paths=source_relative \
  -I proto/ai \
  proto/ai/ai.proto
```

> Never manually edit files inside `internal/pb/` — they are auto-generated by protoc.

The AI client is initialized in `main.go` (currently commented out):
```go
// aiClient := config.ConnectAI()
// defer aiClient.Close()
```
All gRPC call sites inside handlers are also commented with clear markers — search for `// ── AI gRPC call` to find them all.

## Project Structure

```
apps/backend/
├── cmd/
│   └── server/main.go          # Entry point — connects DB, Redis, S3, starts HTTP server
│
├── internal/
│   ├── auth/                   # JWT generation, validation, RequireAuth middleware
│   ├── config/                 # DB (pgxpool), Redis, S3, AI gRPC client connections
│   ├── db/                     # sqlc generated — DO NOT EDIT
│   ├── grpc/ (future)          # Reserved for any Go-side gRPC servers
│   ├── handlers/               # HTTP handlers (auth, user, settings, workspace, maintenance)
│   ├── pb/ai/                  # protoc generated — DO NOT EDIT
│   ├── routes/                 # All route registration
│   └── utils/                  # JSON responses, validation, email, password hashing
│
├── db/
│   ├── migrations/             # SQL migration files (golang-migrate)
│   └── query/                  # sqlc source queries — edit these, then run sqlc generate
│
├── proto/
│   └── ai/ai.proto             # gRPC contract shared with Python AI backend
│
├── sqlc.yaml
├── .air.toml
└── .env
```
